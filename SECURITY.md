# Security Policy

Do not open a public issue for a suspected vulnerability.

For behavior introduced by this fork, use this repository's private
**Report a vulnerability** form under the Security tab. Include the affected
commit, reproduction steps, impact, and whether the issue involves credential
or provider isolation. If private reporting is unavailable, contact the fork
owner privately before sharing details.

For an issue that reproduces in an unmodified `xai-org/grok-build` checkout,
report it through [xAI's HackerOne program](https://hackerone.com/x). Do not
send fork-only OpenAI credentials or logs to the upstream program.

Never include API keys, session tokens, cookies, or unredacted authorization
headers in any report. Revoke an exposed credential with its provider before
continuing the report.
