# Change Records

Change records are the implementation ledger for Bandicot. Use one record for a
coherent feature or migration and update it throughout the work.

## Required Sections

1. Status and owner
2. Problem and intended outcome
3. Scope and non-goals
4. Planned changes and affected paths
5. Engineering rationale and alternatives
6. Pseudocode or state transitions
7. Security and privacy impact
8. Acceptance criteria
9. Verification commands and observed results
10. Implemented changes and deviations
11. Review checkpoints

Valid statuses are `proposed`, `approved`, `implementing`, `verified`,
`shipped`, and `superseded`.
